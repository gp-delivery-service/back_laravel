# Функциональность пополнения фонда админа

## Описание
Добавлена возможность пополнения общего фонда админа через веб-интерфейс. При пополнении средства добавляются как в статичный фонд (`fund`), так и в динамичный фонд (`fund_dynamic`).

## Структура фонда
- **`fund`** - статичный фонд (общий фонд)
- **`fund_dynamic`** - динамичный фонд (доступные средства)
- **`fund_current`** - текущий фонд (вычисляется как fund_dynamic + все деньги в системе)

## Изменения в бэкенде

### 1. Контроллер AdminOperatorsController
- Добавлен метод `topUpFund()` для обработки пополнения фонда
- Валидация входных данных (сумма, тег)
- Обработка ошибок и возврат результата

### 2. Репозиторий AdminFundRepository
- Добавлен метод `topUpFund()` для логики пополнения фонда
- Транзакционная обработка изменений
- Логирование операций в таблицу `gp_admin_fund_logs`

### 3. Маршруты API
- Добавлен POST маршрут `/owner/fund/top-up`

## Изменения во фронтенде

### 1. API контроллер AdminFundController
- Добавлен метод `topUpFund()` для отправки запросов на пополнение

### 2. Компонент TopUpFundDialog
- Новое модальное окно для ввода суммы и тега
- Валидация формы
- Обработка ошибок

### 3. Страница AdminFundPage
- Добавлена кнопка "Пополнить фонд"
- Интеграция с модальным окном
- Обновление данных после успешного пополнения

### 4. Переводы
- Добавлены переводы на русский и туркменский языки

## Использование

1. На странице "Фонд админа" нажмите кнопку "Пополнить фонд"
2. Введите сумму в поле "Сумма (TMT)"
3. При желании добавьте описание операции в поле "Тег"
4. Нажмите "Пополнить фонд"
5. После успешного пополнения данные автоматически обновятся

## Безопасность
- Доступ только для администраторов
- Валидация входных данных
- Транзакционная обработка изменений
- Логирование всех операций

## Логирование
Все операции пополнения фонда записываются в таблицу `gp_admin_fund_logs` с указанием:
- Суммы пополнения
- Старого и нового значения fund_dynamic
- Тега операции
- Пользователя, выполнившего операцию
- Времени выполнения
